Agent Gateway UI Interaction Protocol
Multi-Agent Gateway Protocol Documentation

本节定义前端与多智能体网关（Gateway）交互的 RESTful API 规范。所有路径均以 /api 前缀开头。

API 约定（重点：动词/名词命名 + /api 前缀）：
  • 以下都是动词（动作类请求，建议 POST）： query / upload / submit
  • 以下 GET 都是名词（资源类获取，建议 GET）： viewport / resource / agents / agent / chats / chat
  • 路径示例： /api/query，/api/upload，/api/submit， /api/viewport，/api/resource，/api/agents，/api/agent，/api/chats，/api/chat
定位：前端只对接「多智能体网关（Gateway）」；网关负责路由到具体智能体（Agent），并通过统一的事件流（run/task/message/tool/action 等）把过程与结果回传给前端。
  • requestId：前端生成，用于幂等/重试/链路追踪。
  • chatId：推荐前端生成 UUIDv4（便于离线/重试/跨端关联）；若缺省，网关可创建并在响应与事件中返回。
  • agentKey：可选；不传或传 "auto" 由网关按策略选择默认智能体或最合适智能体。
引用（References）与文本标记（Markers）：
1) 所有引用统一放入 references 数组，并用短 ID：id 进行关联。
2) 在 message 中嵌入标记：#{{refid}}，例如：
"请总结 #{{f1}} 并解释 #{{i1}} 中的流程"
3) 同时建议在 references 显式列出被本次问题依赖的引用（便于渲染、权限、优化）。
4) 前端应优先直接发送带标记的 message；若遗漏标记，网关可在执行前补齐引用提示，但不会改变原始请求结构。
  • id：要求“简短且在本 chat 内不重复”。推荐：文件用 f1,f2,...；图片用 i1,i2,...（或 base36 递增）。
  • type：仅允许 "file" 或 "image"。
  • 工作区 selection / 打开文件等“来源差异”，请放进 reference.meta（例如 meta.origin/meta.filePath/meta.range/meta.textExcerpt）。
数据模型: Reference
Reference
统一引用对象：附件、工作区文件、selection、截图等都用它表达；通过 reference.id 与文本标记/显式 references 关联
  • id: string（短 ID；本 chat 内唯一）例："f1" / "i1"
  • type: "file" | "image"
  • name: string｜可选（展示名；前端请求可省略，服务端生成的引用建议提供）例："requirements.md"
  • mimeType: string（建议总是提供）
  • sizeBytes: number｜可选（上传/本地可得时提供）
  • url: string（网关可访问的地址；或可下载地址）
  • sha256: string｜可选（去重/校验/缓存）
  • sandboxPath: string｜可选（沙箱内可直接访问的路径）例："/workspace/requirements.md"
  • meta: object｜可选（承载“来源差异”与增强信息）
    • origin: "upload" | "workspace" | "clipboard" | "url"｜可选
    • filePath: string｜可选（工作区文件路径）
    • range: {startLine,startCol,endLine,endCol}｜可选（selection 范围）
    • textExcerpt: string｜可选（selection/文件摘要，避免智能体必须先拉全文）
    • caption: string｜可选（图片注释/用户说明）
    • ocrText: string｜可选（图片 OCR 结果，如前端/网关已提取）
请求(动词): QueryRequest（query） /api/query
QueryRequest
发起一次对话请求；当前网关固定以 SSE 返回实时事件流。API: POST /api/query
  • requestId: string（前端生成，用于幂等/重试，前端不填会等于runId）
  • chatId: string(UUID)｜可选；缺省则网关创建 chat
  • agentKey: string｜可选；如 "auto" / "default" / "agent_researcher"
  • teamId: string｜可选；团队/租户路由信息
  • role: "user" | "system" | "assistant" | "developer" | "other"
  • message: string（必填）
  • references: Reference[]｜可选；本次携带的引用（附件/selection/打开文件/截图等）
  • params: object｜可选；结构化输入（input/select/datepicker/switch…）
  • scene: object｜可选；场景页面上下文（精简：以 url 为主）
    • url: string（建议提供）
    • title: string｜可选
  • stream: boolean｜可选；保留作上游兼容字段，当前网关仍固定返回 SSE
  • hidden: boolean｜可选；隐藏本次请求的部分前端展示内容
QueryResponse (current SSE stream)
当前实现不会返回独立 JSON 响应体；POST /api/query 直接返回 text/event-stream。
  • Content-Type: text/event-stream
  • business frames: 业务事件统一使用 event: message，data 为单行 JSON，包含 seq/type/timestamp
  • heartbeat: 传输层保活时会出现 event: heartbeat（无 JSON data）
  • done sentinel: 流结束时追加 data:[DONE]；它不是业务事件，不会进入历史 events
请求(动词): UploadRequest / UploadResponse（upload） /api/upload
UploadRequest
浏览器本地文件一步上传；上传完成后前端可把返回的 upload 映射为 Reference 参与 query。API: POST /api/upload
  • requestId: string（前端生成，用于幂等/重试）
  • chatId: string(UUID)｜可选；缺省则网关创建 chat
  • file: multipart file（必填）
  • sha256: string｜可选
UploadResponse
返回已完成上传的信息；JSON 响应外层仍为统一的 ApiResponse<UploadResponse>。前端可把 upload 映射到本地引用池，并在 query 时通过 #{{refid}} 或 references 使用
  • requestId: string
  • chatId: string(UUID)
  • upload: object（必填）
    • id: string（必填；chat 内短 ID，格式如 r01）
    • type: "file" | "image"
    • name: string
    • mimeType: string
    • sizeBytes: number
    • url: string（必填；上传完成后的资源地址）
    • sha256: string
请求(GET 名词): ViewportRequest / ViewportResponse（viewport） /api/viewport
ViewportRequest
获取工具/渲染对应的视图（表单、UI、HTML 等）；viewportKey 通常来自 tool.start 事件 / ```viewport 内容。API: GET /api/viewport
  • viewportKey: string
ViewportResponse
返回统一 ApiResponse<Object>；data 即视图 payload。本地 HTML viewport 会包装为 {"html":"..."}，其他类型直接透传 payload。
  • code: number
  • msg: string
  • data: any
说明：历史文档中使用 ViewRequest / ViewResponse，此处按统一 GET 名词规范调整为 viewport（即 /api/viewport）。
请求(动词): SubmitRequest / SubmitResponse（submit） /api/submit
SubmitRequest
提交前端工具交互结果。注意：这是公开 HTTP body，不等于流内的 request.submit 事件结构。API: POST /api/submit
  • runId: string
  • toolId: string（必填；当前交互关联的 tool）
  • params: any（必填；表单值/按钮/选择结果等）
SubmitResponse
提交确认；工具执行结果与最终回答走事件流（tool.result/message.*）
  • accepted: boolean
  • status: string
  • runId: string
  • toolId: string
  • detail: string
请求(动词): SteerRequest / SteerResponse（steer） /api/steer
SteerRequest
向进行中的 run 注入新的用户指令。注意：这是公开 HTTP body；流里对应的回看事件是 request.steer，两者不应混用。API: POST /api/steer
  • requestId: string｜可选
  • chatId: string(UUID)｜可选
  • runId: string（必填）
  • steerId: string｜可选（缺省由服务端生成）
  • agentKey: string｜可选
  • teamId: string｜可选
  • message: string（必填）
  • planningMode: boolean｜可选
SteerResponse
引导请求的同步确认；后续处理结果继续走原 SSE 流。
  • accepted: boolean
  • status: string
  • runId: string
  • steerId: string
  • detail: string
请求(动词): InterruptRequest / InterruptResponse（interrupt） /api/interrupt
InterruptRequest
中断进行中的 run。注意：公开 API 名称是 interrupt，但流层不会产生同名的 request.* 事件；中断结果由 run.cancel 表达。API: POST /api/interrupt
  • requestId: string｜可选
  • chatId: string(UUID)｜可选
  • runId: string（必填）
  • agentKey: string｜可选
  • teamId: string｜可选
  • message: string｜可选
  • planningMode: boolean｜可选
InterruptResponse
中断请求的同步确认；实际停止以原 SSE 流中的 run.cancel 为准。
  • accepted: boolean
  • status: string
  • runId: string
  • detail: string
请求(GET 名词): ResourceRequest / ResourceResponse（resource） /api/resource
ResourceRequest
获取资源（含附件/图片）。当前接口直接返回文件内容而非 JSON 包装。API: GET /api/resource
  • file: string（必填；相对 data 根目录的文件路径，兼容 /data/... 形式）
  • download: boolean｜可选（true 时强制下载）
  • t: string｜可选（chat image token）
ResourceResponse
成功时直接返回资源二进制流，失败时返回统一 ApiResponse.failure
  • Content-Type: 由文件名推断 MIME
  • Content-Disposition: 图片默认 inline，其余默认 attachment
  • Cache-Control: private, no-store
请求(GET 名词): AgentsRequest / AgentsResponse（agents） /api/agents
AgentsRequest
获取智能体列表（可用于前端选择或展示）。API: GET /api/agents
  • includeHidden: （暂不实现）boolean｜可选
  • tag: string｜可选（按标签过滤）
AgentsResponse
返回智能体列表
  • agents: object[]（必填）
    • key: string（如 "agent_researcher"）
    • name: string
    • description: string｜可选
    • capabilities: （暂不实现）string[]｜可选
    • meta: （暂不实现）object｜可选
请求(GET 名词): AgentRequest / AgentResponse（agent） /api/agent
AgentRequest
获取某个智能体的内容/详情（配置、说明、能力等）。API: GET /api/agent
  • agentKey: string（必填）
AgentResponse
返回单个智能体详情
  • agent: object（必填）
    • key: string
    • name: string
    • description: string｜可选
    • instructions: string｜可选
    • capabilities: （暂不实现）string[]｜可选
    • meta: （暂不实现）object｜可选
请求(GET 名词): ChatsRequest / ChatsResponse（chats） /api/chats
ChatsRequest
获取聊天列表（历史会话列表）。API: GET /api/chats
  • cursor: string｜可选（分页游标）
  • limit: number｜可选
  • q: string｜可选（按名称/内容检索）
ChatsResponse
返回聊天列表
  • items: object[]（必填）
    • chatId: string(UUID)
    • chatName: string｜可选
    • updatedAt: number(timestamp)｜可选
  • nextCursor: string｜可选
请求(GET 名词): ChatRequest / ChatResponse（chat） /api/chat
ChatRequest
获取聊天内容（会话详情/历史消息/关联 run 等）。API: GET /api/chat
  • chatId: string(UUID)（必填）
  • includeEvents: boolean｜可选（是否返回事件流快照）
  • cursor: string｜可选（分页消息）
  • limit: number｜可选
ChatResponse
返回聊天内容（结构可按产品需要扩展；不改变原事件模型，仅补充 GET 入口）
  • chatId: string(UUID)
  • chatName: string｜可选
  • chatImageToken: string｜可选（用于访问当前 chat 作用域内图片资源）
  • rawMessages: object[]｜可选（若 includeRawMessages=true，可返回大模型原始消息）
  • plan: object｜可选（chat 当前最新计划状态，形如 {planId,tasks}）
  • artifact: object｜可选（chat 当前最新产物聚合状态，形如 {items:[...]}）
  • references: object[]｜可选（query 引用与 chat 资产聚合后的引用池）
  • events: BaseEvent[]（历史事件列表； start/delta/args/end -> snapshot，保留 params/result；plan.update / artifact.publish 不再出现在历史 events 中）
CHAT
chatId start/update
PLAN
planId, chatId create/update
TaskElement 1 elementId
TaskElement 2 elementId
RUN
runId, chatId start/complete/error/cancel
TASK
taskId, runId start/complete/cancel/fail
REASONING
reasoningId, runId, [taskId] start/delta/end/snapshot
CONTENT
contentId, runId, [taskId] start/delta/end/snapshot
TOOL
toolId, runId, [taskId] start/args/end/result/snapshot
ACTION
actionId, runId, [taskId] start/args/end/param/result/snapshot
ARTIFACT
artifactId, runId, chatId publish
SOURCE(未实现)
sourceId, runId, [taskId] snapshot
Event Documentation
基类: Base Event
Base
所有事件类的抽象基类，定义了事件的基本结构
  • seq: 唯一序列号（已实现；实时业务事件统一包含）
  • type: 事件类型 (枚举值)
  • timestamp: 事件创建时间戳
  • rawEvent: [调试用]原始事件对象
这里只列会真实出现在 SSE 流里的 request.* 事件；公开 HTTP API 不要求和流事件形成 1:1 命名映射，例如 POST /api/interrupt 不会生成对应的 request 类事件。
请求: Request Events（新增补充）
RequestQuery
请求动作事件：request.query（动词：query；对应 API: POST /api/query）
  • type: "request.query"
  • requestId: 前端生成，用于幂等/重试/链路追踪
  • chatId: 聊天会话 ID（可由前端生成或由网关创建后回传）
  • agentKey: 可选；如 "auto" / "default" / "agent_researcher"
  • teamId: string｜可选
  • role: "user" | "system" | "assistant" | "developer" | "other"
  • message: string（建议与 QueryRequest 对齐）
  • references: Reference[]｜可选
  • params: object｜可选
  • scene: object｜可选（{url,title}）
  • stream: boolean｜可选；当前网关仍固定返回 SSE
  • hidden: boolean｜可选
RequestUpload
请求动作事件：request.upload（对应 API: POST /api/upload）。这是统一事件记录中的内部对象；公开 HTTP 请求本身是 multipart/form-data，不是 JSON upload 对象。
  • type: "request.upload"
  • requestId: 前端生成
  • chatId: chatId（可选；缺省则网关创建）
  • upload: object（内部事件载荷）
    • type: "file" | "image"
    • name: string
    • sizeBytes: number
    • mimeType: string
    • sha256: string｜可选
RequestSubmit
请求动作事件：request.submit（对应 API: POST /api/submit）。这是运行流里的回看事件，不等于公开 HTTP body。
  • type: "request.submit"
  • requestId: 前端生成
  • chatId: string(UUID)
  • runId: string
  • toolId: string（提交给哪个工具）
  • payload: any（表单值/按钮/选择结果等）
  • viewId: string｜可选
RequestSteer
请求动作事件：request.steer（对应 API: POST /api/steer）。这是运行流里的回看事件，用于记录运行中注入的追加指令；它不是 /api/steer 的完整 HTTP body schema。
  • type: "request.steer"
  • requestId: string｜可选
  • chatId: string(UUID)
  • runId: string
  • steerId: string
  • message: string
  • role: 固定为 "user"
传输: Transport Events
Heartbeat
SSE 传输层保活事件；由注释帧标准化而来，不属于业务 JSON 事件。
  • wire format: event: heartbeat
  • data: 无
Done Sentinel
流结束时追加的传输层终止帧，不是业务事件，也不会写入历史 events。
  • wire format: data:[DONE]
  • data: 非 JSON 文本
会话: Chat Events
ChatStart
开始聊天会话的事件
  • chatId: 聊天会话的唯一标识
  • chatName: 聊天会话的名称
当前实时 SSE 只发会话开始事件；名称变更不会额外输出一条独立的实时会话更新事件。
计划: Plan Events
PlanUpdate
更新任务计划的事件
  • planId: 计划的唯一标识
  • chatId: 关联的聊天会话ID
  • plan: 计划列表，包含多个任务对象
当前实时 SSE 统一使用计划更新事件表达首次创建与后续修改，不再拆成单独“创建”事件。
运行: Run Events
RunStart
运行开始的事件
  • runId: 运行实例的唯一标识
  • chatId: 关联的聊天会话ID
  • agentKey: 运行的智能体KEY
RunComplete
运行完成的事件
  • runId: 完成的运行实例ID
  • finishReason: string（缺省为 end_turn）
RunCancel
运行被取消/中断时的实时事件。常见来源是 POST /api/interrupt；当前流层不会发独立的 interrupt request 事件，而是直接以 run.cancel 表达中断结果。
  • runId: 被取消的运行实例ID
RunError
运行出错的事件
  • runId: 出错的运行实例ID
  • error: 错误信息
任务: Task Events
TaskStart
任务开始执行的事件
  • taskId: 任务的唯一标识
  • runId: 所属运行实例ID
  • taskName: 任务名称
  • description: 任务描述
TaskComplete
任务完成的事件
  • taskId: 完成的任务ID
TaskCancel
任务被取消的事件
  • taskId: 被取消的任务ID
TaskFail
任务失败的事件
  • taskId: 失败的任务ID
  • error: 错误信息
推理: Reasoning Events
ReasoningStart
推理开始事件
  • reasoningId: 推理的唯一标识
  • runId: 运行实例ID
  • taskId: 可选，任务ID
ReasoningDelta
推理增量更新事件
  • reasoningId: 推理的唯一标识
  • delta: 推理的增量部分
ReasoningEnd
推理结束事件
  • reasoningId: 推理的唯一标识
ReasoningSnapshot
推理快照事件（主要用于历史记录；不是当前 /api/query 实时流主路径）
  • reasoningId: 推理的唯一标识
  • runId: 运行实例ID
  • taskId: 可选，任务ID
  • text: 推理的文字
正文: Content Events
ContentStart
正文开始事件
  • contentId: 正文的唯一标识
  • runId: 运行实例ID
  • taskId: 可选，任务ID
ContentDelta
正文增量更新事件
  • contentId: 正文的唯一标识
  • delta: 正文的增量部分
ContentEnd
正文结束事件
  • contentId: 正文的唯一标识
ContentSnapshot
正文快照事件（主要用于历史记录；不是当前 /api/query 实时流主路径）
  • contentId: 正文的唯一标识
  • runId: 运行实例ID
  • taskId: 可选，任务ID
  • text: 正文的文字
工具: Tool Events
ToolStart
工具开始事件；前端工具在最终输出中会补充 viewport 相关字段
  • toolId: 工具的唯一标识
  • runId: 运行实例ID
  • toolName: 工具名称
  • taskId: 可选，任务ID
  • toolType: 工具类型｜可选；前端工具常见为 html/qlc/dqlc
  • toolLabel: string｜可选；展示名
  • toolDescription: string｜可选；展示描述
  • viewportKey: string｜仅前端工具会出现
  • toolTimeout: number(ms)｜仅前端工具会出现
ToolArgs
工具渲染参数更新事件
  • toolId: 工具的唯一标识
  • delta: 工具参数的增量部分
  • chunkIndex: number；同一 toolId 下的分片序号
ToolEnd
工具渲染结束事件
  • toolId: 工具的唯一标识
ToolSnapshot
工具快照事件（主要用于历史记录；不是当前 /api/query 实时流主路径）
  • toolId: 工具的唯一标识
  • runId: 运行实例ID
  • toolName: 工具名称
  • taskId: 可选，任务ID
  • toolType: 工具类型｜可选
  • toolLabel: string｜可选
  • toolDescription: string｜可选
  • viewportKey: string｜前端工具可能出现
  • toolTimeout: number(ms)｜前端工具可能出现
  • arguments: 工具参数快照
ToolResult
工具运行结果事件
  • toolId: 工具的唯一标识
  • result: 工具运行结果
前端交互提交走 POST /api/submit；运行流里会记录为 request.submit，不会额外发一条独立的“工具参数事件”。
产物: Artifact Events
ArtifactPublish
独立产出物发布事件；每个产物对应一条 artifact.publish。
  • artifactId: 产出物唯一标识
  • chatId: 关联 chatId
  • runId: 关联 runId
  • artifact: 精简产物对象，仅包含 type/name/mimeType/sizeBytes/url/sha256
一次调用工具 _artifact_publish_ 时，artifacts[] 里有几个产物，就会出现几条事件 artifact.publish。
常见顺序是 tool.start -> tool.args -> tool.end -> tool.result -> artifact.publish；如果一次调用里有多个产物，tool.result 后会连续出现多条 artifact.publish。
动作: Action Events
ActionStart
动作渲染开始事件
  • actionId: 动作的唯一标识
  • runId: 运行实例ID
  • actionName: 动作名称
  • taskId: 可选，任务ID
  • description: 动作描述
ActionArgs
动作渲染参数更新事件
  • actionId: 动作的唯一标识
  • delta: 动作参数的增量部分
ActionEnd
动作渲染结束事件
  • actionId: 动作的唯一标识
ActionSnapshot
动作快照事件（主要用于历史记录；不是当前 /api/query 实时流主路径）
  • actionId: 动作的唯一标识
  • runId: 运行实例ID
  • actionName: 动作名称
  • taskId: 可选，任务ID
  • description: 动作描述
  • arguments: 动作参数
ActionResult
动作执行结果事件
  • actionId: 动作的唯一标识
  • result: 动作的结果（response）
来源: Source Events
SourceSnapshot
源信息块的快照事件；当前仍未实现实时 SSE 输出
  • icon: 来源图标
  • title: 来源标题
  • url: 来源URL地址
用例以「公开 API 请求」+「对应 SSE 事件流」为最小闭环展示；基础场景通常从 POST /api/query 开始，运行中也可以继续发送 POST /api/steer / POST /api/interrupt。注意： reasoning.delta/tool.args/content.delta 都是流式“吐字”（会拆成多段）； 业务事件统一采用 event: message + data: {"seq":...,"type":"...","timestamp":...} 一行 JSON； 流结束时会追加传输层终止帧 data:[DONE]。
